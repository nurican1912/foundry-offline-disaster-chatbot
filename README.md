# Offline Emergency Assistant (Local RAG)

A RAG-based emergency assistant that runs **entirely on the local device** for major disaster situations (earthquakes, etc.) where internet infrastructure has collapsed. The system is grounded **only** in verified disaster/first-aid guides; if the similarity score is too low it refuses to answer instead of guessing (**zero-hallucination** principle).

> For detailed architecture, decisions, and the module roadmap, see [`EMERGENCY_RAG_ASSISTANT_PLAN.md`](EMERGENCY_RAG_ASSISTANT_PLAN.md) (in Turkish).

## Tech Stack

- **Language/Framework:** Python + FastAPI (planned)
- **Local model runtime:** Microsoft Foundry Local (SDK)
- **Chat model:** `qwen2.5-1.5b` (CPU) · **Embedding:** `qwen3-embedding-0.6b` (CPU, 1024-dim)
- **Data layer:** SQLite (stores both text and embeddings)

## Architecture (5 layers, single machine)

```
Frontend → FastAPI → RAG Pipeline → SQLite (chunks + embeddings) → Foundry Local (embed + chat)
```

## Setup

Prerequisite: [Microsoft Foundry Local](https://learn.microsoft.com/azure/ai-foundry/foundry-local/) must be installed.

```bash
python -m venv .venv
# Windows:
.\.venv\Scripts\Activate.ps1
# Linux/Mac:
# source .venv/bin/activate

pip install -r requirements.txt
```

Optionally copy `.env.example` to `.env` and adjust settings (optional; defaults are fine).

## Building the knowledge base (ingestion)

Builds the searchable database from the source documents in `docs/raw/`:

```python
from app.data.database import init_db, save_chunks
from app.rag.chunking import get_all_chunks
from app.rag.ingest import embed_pending_chunks

init_db("data/emergency.db")
save_chunks("data/emergency.db", get_all_chunks("docs/raw"))   # writes 420 chunks
embed_pending_chunks("data/emergency.db")                       # embeds each chunk into a 1024-dim vector
```

> `data/emergency.db` is **not** committed — it is regenerated by the steps above.

## Project structure

```
app/
  config.py        # pydantic-settings configuration
  ai/embeddings.py # embedding provider (Foundry Local)
  data/database.py # SQLite schema + read/write
  rag/
    chunking.py    # splits documents into chunks
    ingest.py      # chunk → embed → DB bridge
    retrieval.py   # cosine similarity + top-k + threshold (in progress)
  api/             # FastAPI (planned)
docs/raw/          # verified source documents (knowledge base)
```
